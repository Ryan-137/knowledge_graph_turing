# 图灵知识图谱数据预处理实验报告

## 1. 实验目的

本实验的目标是对 `source` 文件夹中的图灵相关原始资料进行统一预处理，为后续的实体抽取、关系抽取和知识图谱构建提供干净、结构化的语料。

原始资料主要来源于网页存档和 PDF，包括：

* BBC News
* The Guardian
* Wikiwand
* 百度百科
* 知乎专栏
* 纪念性网页资料
* PDF 文档

这些数据存在格式不统一、网页噪声多、正文与导航混杂等问题，因此需要先完成清洗和标准化。

---

## 2. 实验环境

本实验在当前项目目录下完成，核心脚本为 `preprocess.py`。

使用到的主要工具和库：

* Python
* `BeautifulSoup`
* `lxml`
* `json`
* `pathlib`
* `re`

说明：

* 当前环境中 `BeautifulSoup` 和 `lxml` 可正常使用。
* 当前环境中未安装 `pypdf` 或 `PyPDF2`，因此 PDF 文件暂时只输出占位说明，不会中断整个预处理流程。

---

## 3. 实验数据

输入目录为：

```text
source/
```

脚本支持处理以下类型文件：

```text
.html
.htm
.txt
.md
.pdf
```

同时自动跳过网页资源目录，例如：

```text
*_files/
```

这样可以避免把网页附带的脚本、样式、图片资源误当成文本语料。

---

## 4. 实验思路

本次预处理围绕“从原始网页中尽量提取可用于知识抽取的正文”展开，主要分为以下几个步骤：

### 4.1 文件筛选

遍历 `source` 目录，筛选支持的文件格式，只保留真正可能包含文本内容的源文件，跳过资源文件夹和无关文件。

### 4.2 标题提取

对 HTML 文件优先从以下位置提取标题：

* `<title>`
* `og:title`
* `twitter:title`
* 页面首个标题标签

对于无法稳定提取正文的网站，至少保留正确标题，保证后续语料可追溯。

### 4.3 网页正文提取

对 HTML 文件，首先删除以下明显非正文标签：

* `script`
* `style`
* `noscript`
* `svg`
* `form`
* `button`
* `header`
* `footer`
* `nav`
* `aside`
* `iframe`

随后使用两种方式提取正文：

1. 优先匹配常见正文容器，如 `article`、`main`、`.content`、`.RichText` 等。
2. 如果没有明显正文容器，则根据文本密度、段落数量、链接密度等规则选择最可能的正文区域。

### 4.4 特殊站点处理

不同网站的 HTML 结构差异较大，因此加入了简单的站点特化规则：

* 百度百科：优先提取页面 `description` 摘要，避免把目录、参考资料、图片说明等大量噪声当作正文。
* 其他站点：按通用网页正文抽取规则处理。

### 4.5 文本清洗

提取出的文本继续进行清洗，包括：

* 去除重复行
* 去除脚本/CSS 风格文本
* 去除明显资源链接和无意义短行
* 统一空白字符
* 去除部分目录式短标题和参考文献列表边缘噪声

### 4.6 分句

为了便于后续实体识别和关系抽取，脚本会进一步对清洗后的文本做分句处理，生成句子列表。

分句依据主要包括：

* `。`
* `！`
* `？`
* `；`
* 英文句号加空格后的句边界

### 4.7 结构化输出

预处理结果统一写入 `preprocessed` 目录，输出为：

* `documents.json`：所有文档的结构化结果
* `summary.json`：统计信息
* `texts/*.txt`：每篇文档对应的清洗后纯文本

## 5. 核心实现

本次实验中，`preprocess.py` 主要实现了以下功能模块：

### 5.1 文档扫描

扫描 `source` 下的全部文件，筛选出支持的文档类型。

### 5.2 HTML 抽取

通过 `BeautifulSoup` 解析网页结构，删除噪声标签，并选择最佳正文容器提取文本。

### 5.3 PDF 兼容处理

若环境中安装了 `pypdf` 或 `PyPDF2`，则自动抽取 PDF 文本；若未安装，则保留文件名并给出“跳过 PDF 抽取”的说明。

### 5.4 清洗与去重

使用正则表达式和规则过滤完成去噪，提升文本纯度。

### 5.5 数据组织

每篇文档最终组织为如下结构：

```json
{
  "doc_id": "文档唯一标识",
  "title": "标题",
  "source_path": "原始文件路径",
  "source_type": "html/pdf/text",
  "clean_text": "清洗后的正文",
  "sentences": ["句子1", "句子2"],
  "char_count": 1234
}
```

## 7. 实验分析

本次实验说明，网页类知识图谱语料预处理不能简单使用 `get_text()` 一次性抽取，否则容易把如下内容一起抓出：

* 导航栏
* 标签云
* 目录
* 参考资料
* 页脚信息
* JS/CSS 片段

因此必须结合网页结构、文本密度和规则过滤做多层清洗。

另外，不同来源的数据风格差异明显：

* 新闻网站的正文结构通常较清晰，较容易提取。
* 百科类网站信息密集，但目录和引用噪声很多，需要专门适配。
* 本地保存网页可能包含插件注入内容，也会干扰抽取结果。

这说明在构建知识图谱时，预处理阶段本身就是一个非常重要的质量控制环节。

## 8. 存在问题

当前预处理仍有以下不足：

* 个别网页仍残留少量章节标题或短标签文本
* 尚未做中英文实体名称规范化
* 尚未做时间格式统一
* 尚未做人名别称对齐，例如：
  * Alan Turing
  * Alan Mathison Turing
  * 艾伦·图灵
  * 阿兰·图灵

这些问题会在后续实体对齐与知识融合阶段继续处理。
